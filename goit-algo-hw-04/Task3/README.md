# Порівняння алгоритмів сортування

Порівняльна таблиця результатів сортування масивів випадкових чисел:

| Алгоритм | 100 | 1000 | 5000 | 10000 |
| --- | ---: | ---: | ---: | ---: |
| Сортування вставками | 0.000155 | 0.026647 | 0.485506 | 2.000595 |
| Сортування злиттям | 0.000135 | 0.008087 | 0.011647 | 0.024325 |
| Timsort | 0.000010 | 0.000222 | 0.000675 | 0.001243 |

Висновок: найшвидше у всіх тестах спрацював Timsort, тобто вбудоване сортування Python. Сортування злиттям теж показало хороший результат, особливо на більших масивах. Сортування вставками на малому масиві ще працює нормально, але зі збільшенням кількості елементів стає набагато повільнішим. Тому в реальних задачах краще користуватися вбудованим `sorted`, а власні алгоритми писати більше для навчання і розуміння принципів роботи сортування.
